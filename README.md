## 系统简介

智慧停车场管理系统是一套集 **AI 车牌识别、车辆进出管理、在线模拟支付、2.5D 车位可视化** 于一体的停车场管理平台。

系统分为两类用户：

| 角色 | 主要职责 |
|------|---------|
| **管理员** | 操作车辆进出、查看数据统计、管理车位与收费规则 |
| **普通用户** | 查询本人车牌的停车记录、在线支付停车费 |

## 方式一：Docker 一键运行（推荐，最简单）

> 适合所有人，无需安装 Java / Python / MySQL / Node.js，只需要安装 Docker。

### 第一步：安装 Docker Desktop

1. 打开浏览器，访问：**https://www.docker.com/products/docker-desktop/**
2. 点击页面上的 **"Download for Windows"** 按钮下载安装包
3. 双击下载好的 `.exe` 文件，一路点击 **"下一步"** 安装
4. 安装完成后，**重启电脑**
5. 重启后桌面上会出现 Docker Desktop 图标，双击打开它
6. 等待 Docker Desktop 启动完毕（底部状态栏显示绿色小圆点即为就绪）

> 如果提示需要安装 WSL2，按照提示点击安装即可，然后重启。

---

### 第二步：获取镜像文件

**如果你有 `smart-parking.tar` 文件（通过U盘/网盘获取）：**

跳过此步，直接进入第三步。

**如果你想自己构建（需要先 clone 代码）：**

1. 安装 Git：https://git-scm.com/download/win，一路默认安装
2. 右键桌面，选择 **"Git Bash Here"**，输入：
   ```bash
   git clone https://github.com/jinweixia03/Smart-Parking.git
   cd Smart-Parking
   ```
3. 双击运行项目里的 `build-image.bat`，等待 10-20 分钟自动构建
4. 构建完成后会生成 `smart-parking.tar` 文件

---

### 第三步：加载并运行镜像

1. 按下键盘 `Win + R`，输入 `cmd`，按回车，打开命令提示符
2. 输入以下命令（注意把路径换成你的 tar 文件实际位置）：

```cmd
docker load -i C:\Users\你的用户名\Desktop\smart-parking.tar
```

> 例如文件在桌面，用户名是 zhang，就输入：
> `docker load -i C:\Users\zhang\Desktop\smart-parking.tar`

3. 加载完成后，输入以下命令启动：

```cmd
docker run -d -p 3000:3000 --name parking --restart always smart-parking:latest
```

4. 等待约 **30 秒**，然后打开浏览器，访问：

```
http://localhost:3000
```

看到登录页面说明启动成功。

---

### 常用 Docker 命令

```cmd
# 查看运行状态
docker ps

# 停止服务
docker stop parking

# 再次启动（下次开机后用这个，无需重新 load）
docker start parking

# 查看日志
docker logs parking

# 彻底删除容器（数据会丢失）
docker rm -f parking
```

## 方式二：本地手动运行（开发者）

> 适合需要修改代码的开发者。需要逐步安装所有环境。

### 环境要求

| 组件 | 版本 | 说明 |
|------|------|------|
| Java JDK | 17 | 运行后端 |
| Maven | 3.9 | 构建后端 |
| Node.js | 18+ | 运行前端 |
| Python | 3.10 | 运行算法服务 |
| MySQL | 8.0 | 数据库 |
| MiniConda | 最新 | 管理 Python 环境 |

---

### 第一步：安装 Java 17

1. 访问：https://adoptium.net/
2. 选择 **Temurin 17 (LTS)**，点击下载 Windows x64 `.msi` 安装包
3. 双击安装，**安装时勾选 "Set JAVA_HOME variable"**（重要！）
4. 安装完成后，打开命令提示符（Win+R → cmd），输入：
   ```cmd
   java -version
   ```
   看到 `openjdk version "17..."` 说明安装成功

---

### 第二步：安装 Maven

1. 访问：https://maven.apache.org/download.cgi
2. 下载 **Binary zip archive**（如 `apache-maven-3.9.x-bin.zip`）
3. 解压到一个固定目录，例如 `D:\Environments\maven\apache-maven-3.9.x`
4. 配置环境变量：
   - 右键 "此电脑" → 属性 → 高级系统设置 → 环境变量
   - 在 **系统变量** 中，找到 `Path`，点击编辑
   - 点击新建，输入你的 Maven bin 目录，例如：`D:\Environments\maven\apache-maven-3.9.x\bin`
   - 一路点确定保存
5. 重新打开命令提示符，输入：
   ```cmd
   mvn -version
   ```
   看到版本号说明安装成功

---

### 第三步：安装 Node.js

1. 访问：https://nodejs.org/
2. 下载 **LTS 版本**（长期支持版），点击下载
3. 双击 `.msi` 安装包，一路默认安装（**不要修改安装路径**）
4. 安装完成后，打开新的命令提示符，输入：
   ```cmd
   node -v
   npm -v
   ```
   两个都显示版本号说明安装成功

---

### 第四步：安装 MiniConda（Python 环境管理）

1. 访问：https://docs.conda.io/en/latest/miniconda.html
2. 下载 **Miniconda3 Windows 64-bit**
3. 双击安装，安装时：
   - **勾选 "Add Miniconda3 to my PATH"**（重要！）
   - 安装路径建议用默认路径或 `D:\Environments\MiniConda3`
4. 安装完成后，打开新的命令提示符，输入：
   ```cmd
   conda -V
   ```
   显示版本号说明安装成功
5. 创建并激活 Python 环境：
   ```cmd
   conda create -n yolo python=3.10 -y
   conda activate yolo
   ```
6. 安装算法依赖（在 yolo 环境中）：
   ```cmd
   cd 项目目录\algorithm
   pip install torch==2.1.0 torchvision==0.16.0 --index-url https://download.pytorch.org/whl/cu118
   pip install -r requirements.txt
   ```

---

### 第五步：安装 MySQL 8.0

1. 访问：https://dev.mysql.com/downloads/mysql/
2. 选择 **MySQL Community Server 8.0**，下载 Windows 安装包
3. 双击安装，选择 **Developer Default**，一路下一步
4. 设置 root 密码时，**密码设置为 `123456`**（与项目配置一致）
5. 安装完成后，打开命令提示符，输入：
   ```cmd
   mysql -u root -p
   ```
   输入密码 `123456`，能进入 MySQL 说明安装成功
6. 初始化数据库（在 MySQL 命令行中）：
   ```sql
   CREATE DATABASE smart_parking CHARACTER SET utf8mb4;
   USE smart_parking;
   SOURCE D:/你的项目路径/backend/src/main/resources/db/sp.sql;
   EXIT;
   ```

---

### 第六步：修改 start-all.bat 路径配置

打开项目根目录的 `start-all.bat`，用记事本或 VSCode 打开，修改开头的路径变量：

```bat
REM 根据你的实际安装路径修改以下变量

set "BACKEND_DIR=D:\1_Source\BSYCode\backend"      ← 改成你的项目路径
set "FRONTEND_DIR=D:\1_Source\BSYCode\frontend"    ← 改成你的项目路径
set "ALGORITHM_DIR=D:\1_Source\BSYCode\algorithm"  ← 改成你的项目路径
set "MAVEN_CMD=D:\Environments\maven\apache-maven-3.9.10\bin\mvn.cmd"  ← 改成你的 Maven 路径
set "CONDA_DIR=D:\Environments\MiniConda3"         ← 改成你的 MiniConda 安装路径
set "CONDA_ENV=yolo"                                ← 保持不变
```

**如何找到你的项目路径：**
- 在文件资源管理器中打开项目文件夹
- 点击地址栏，复制完整路径，例如 `C:\Users\zhang\Desktop\Smart-Parking`

**如何找到 Maven 路径：**
- 找到你解压 Maven 的文件夹，复制 `bin\mvn.cmd` 的完整路径

**如何找到 MiniConda 路径：**
- 默认安装在 `C:\Users\你的用户名\miniconda3`
- 如果选择了自定义路径，找到安装时选择的目录

---

### 第七步：启动所有服务

修改好路径后，双击运行 `start-all.bat`，等待所有服务启动。

窗口显示以下内容说明启动成功：
```
[OK] All services started!
Frontend:  http://localhost:3000
Backend:   http://localhost:8080/api
Algorithm: http://localhost:5000/api/health
```

打开浏览器访问：**http://localhost:3000**

## 一、账号与登录

### 1.1 登录系统

1. 打开浏览器（推荐 Chrome 或 Edge），在地址栏输入 `http://localhost:3000`，按回车
2. 页面显示登录界面，在**用户名**输入框输入账号，在**密码**输入框输入密码
3. 点击蓝色**登录**按钮
4. 登录成功后：
   - **管理员账号** → 自动跳转到数据大屏页面
   - **普通用户账号** → 自动跳转到"我的停车记录"页面

> 如果出现"用户名或密码错误"提示，请检查大小写是否正确。

---

### 1.2 注册新账号（仅普通用户）

管理员账号由系统预设，普通用户可以自行注册。

1. 在登录页面，点击登录按钮下方的**注册账号**链接
2. 填写以下信息：
   - **用户名**：只能包含字母、数字、下划线，长度 4-20 位
   - **密码**：长度至少 6 位
   - **确认密码**：与密码一致
   - **手机号**：用于找回密码（填写真实手机号）
3. 点击**注册**按钮
4. 注册成功后会弹出提示，点击"去登录"即可使用新账号登录

---

### 1.3 忘记密码

1. 在登录页面，点击**忘记密码**链接
2. 填写以下信息：
   - **用户名**：注册时使用的用户名
   - **手机号**：注册时填写的手机号
   - **新密码**：设置新的登录密码
3. 点击**确认重置**，成功后使用新密码登录即可

> 注意：用户名和手机号必须与注册时完全一致，否则无法重置。

---

### 1.4 修改个人信息与密码

登录后，点击页面**右上角的用户名或头像**，下拉菜单中选择：

- **个人信息**：可修改绑定的手机号
- **修改密码**：需要输入当前密码，再输入两次新密码确认

修改完成后点击保存，**立即生效**，下次登录使用新密码。

---

### 1.5 退出登录

点击页面右上角用户名，下拉菜单选择**退出登录**，系统清除登录状态并跳转回登录页。

## 二、管理员功能详解

> 以下功能仅管理员账号可见，普通用户登录后不会显示这些菜单。

---

### 2.1 数据大屏

**入口：** 左侧导航栏 → 数据大屏

数据大屏是系统首页，展示停车场**实时运营状态**，所有数据每 **30 秒**自动刷新一次。

#### 顶部统计卡片（共 7 块）

| 卡片名称 | 含义 | 备注 |
|---------|------|------|
| 当前停车数 | 此刻场内停放的车辆总数 | 实时数据 |
| 总车位数 | 停车场全部车位总数 | 固定值 |
| 空闲车位 | 当前未被占用的车位数 | 总数 - 占用数 |
| 占用率 | 已占用车位 ÷ 总车位 × 100% | 百分比显示 |
| 今日入场 | 今天 00:00 至今入场的车辆数 | 每天重置 |
| 今日出场 | 今天 00:00 至今出场的车辆数 | 每天重置 |
| 今日收入 | 今天已完成支付的停车费合计 | 单位：元 |

#### 图表区域

页面下半部分有三种统计图表，可点击切换：

- **分时统计**：以每小时为单位，显示今日各时段入场数和出场数的折线图，用于分析高峰时段
- **日统计**：显示近 7 天每天的入场数、出场数和收入趋势，用于分析周规律
- **区域统计**：以饼图或柱状图展示 A、B、C、D 各区域当前车位占用比例

---

### 2.2 车牌检测（核心功能）

**入口：** 左侧导航栏 → 车牌检测

这是管理员**每天使用最频繁**的页面，负责处理所有车辆的入场和出场登记。

#### 页面布局

页面分为左右两栏：
- **左栏**：图片上传区、识别信息显示、车牌号输入框、操作选择按钮
- **右栏**：AI 检测结果图（含检测框标注）

---

#### 完整操作流程：车辆入场

**场景：** 有一辆车进入停车场，需要登记入场。

**第一步：上传车牌图片**

- 点击左侧**上传图片**区域（虚线框内），打开文件选择窗口
- 也可以直接将图片**拖拽**到虚线框内
- 支持格式：JPG、JPEG、PNG
- 图片建议清晰、车牌完整可见，不要过度模糊或遮挡

**第二步：等待 AI 自动识别**

- 图片上传后，系统自动调用 AI 算法识别车牌，无需手动点击
- 识别过程中左侧显示**扫描线动画**，提示"AI 正在识别车牌..."
- 识别通常在 **1-15 秒**内完成（CPU 模式较慢，GPU 模式较快）

**第三步：查看识别结果**

识别完成后：

- **右侧**：显示原图，车牌位置用**绿色矩形框**标出，框内标注识别到的车牌号
- **左侧信息区**显示：
  - 识别车牌：AI 读取的车牌号码，如 `皖AH2795`
  - 定位置信度：AI 找到车牌位置的准确率，如 `98.5%`（越高越准）
  - 识别置信度：AI 读取车牌号的准确率，如 `95.2%`（越高越准）
  - 检测耗时：本次识别用时，如 `1253ms`

如果识别失败（图片无法识别出车牌），左侧会显示红色提示"未检测到车牌，请手动输入"，右侧显示原图无框。

**第四步：确认或修改车牌号**

- 识别成功后，**车牌号自动填入**下方输入框
- 仔细核对识别结果是否正确（重点检查容易混淆的字符，如 `0` 和 `O`、`1` 和 `I`）
- 如有错误，直接在输入框内**修改**即可
- 如识别失败，在输入框**手动输入**完整车牌号（格式如 `皖AH2795`）

**第五步：选择"车辆入场"**

在操作类型选择中，点击**车辆入场**单选按钮（默认已选中）。

**第六步：点击"执行操作"**

点击蓝色**执行操作**按钮，系统处理入场登记：

- **成功**：弹窗显示
  ```
  入场成功
  车牌号：皖AH2795
  分配车位：A-369
  入场时间：2026-03-20 10:30:00
  ```
  点击确认关闭弹窗，准备处理下一辆车

- **失败**：可能原因
  - "该车牌已在场内" → 该车牌已有入场记录未出场，核实是否同一辆车
  - "停车场已满" → 所有车位均已占用，无法入场
  - "车牌号不能为空" → 未填写车牌号

---

#### 完整操作流程：车辆出场

**场景：** 有一辆车要离开停车场，需要办理出场手续。

**第一步至第四步** 与入场相同（上传图片 → AI 识别 → 确认车牌号）

**第五步：选择"车辆出场"**

在操作类型选择中，点击**车辆出场**单选按钮。

**第六步：点击"执行操作"**

点击蓝色**执行操作**按钮，系统计算费用并完成出场登记：

- **成功 - 免费情况**：
  ```
  出场成功
  车牌号：皖AH2795
  停车时长：8分钟
  停车费用：0元（免费）
  ```
  停车不超过 15 分钟（免费时长）自动免费，放行即可。

- **成功 - 需要付费情况**：
  ```
  出场成功
  车牌号：皖AH2795
  停车时长：2小时35分钟
  停车费用：11元
  支付状态：未支付
  ```
  告知车主前往"我的停车记录"页面，输入车牌号后可在线支付。

- **失败**：可能原因
  - "未找到该车牌的停车记录" → 该车牌没有入场记录，可能未登记或车牌有误
  - 请核实车牌号是否正确

---

#### 不使用 AI 识别直接手动操作

如果算法服务未启动，或者图片质量太差无法识别，可以完全跳过图片上传：

1. 直接在**车牌号输入框**中手动输入车牌号
2. 选择操作类型（入场/出场）
3. 点击执行操作

功能完全正常，AI 识别只是辅助录入，不是必须步骤。

---

### 2.3 停车记录管理

**入口：** 左侧导航栏 → 停车记录

## 四、收费规则说明

### 计费公式

```
第一步：判断是否免费
  如果 停车时长 ≤ 免费时长（默认15分钟）
      → 费用 = 0 元，标记为"免费"，流程结束

第二步：计算基础费用
  计费小时数 = 向上取整(停车分钟数 ÷ 60)

  如果 计费小时数 = 1
      基础费用 = 首小时费用（默认5元）
  如果 计费小时数 > 1
      基础费用 = 首小时费用 + (计费小时数 - 1) × 超出每小时费用

第三步：应用每日封顶
  计算经过的天数 = 计费小时数 ÷ 24（取整）
  剩余小时数 = 计费小时数 mod 24

  最终费用 = 完整天数 × 每日封顶费用 + min(剩余小时费用, 每日封顶费用)
```

### 计费示例（默认规则：免费15分，首小时5元，超出3元/时，封顶50元）

| 停车时长 | 计算过程 | 应缴费用 |
|---------|---------|---------|
| 5 分钟 | 未超免费时长 | **0 元（免费）** |
| 15 分钟 | 恰好等于免费时长 | **0 元（免费）** |
| 16 分钟 | 超出免费，向上取整为 1 小时 | **5 元** |
| 30 分钟 | 超出免费，1 小时 | **5 元** |
| 60 分钟 | 1 小时 | **5 元** |
| 61 分钟 | 向上取整为 2 小时：5 + 1×3 | **8 元** |
| 90 分钟 | 2 小时：5 + 1×3 | **8 元** |
| 3 小时 | 5 + 2×3 | **11 元** |
| 5 小时 | 5 + 4×3 | **17 元** |
| 10 小时 | 5 + 9×3 = 32 | **32 元** |
| 18 小时 | 5 + 17×3 = 56 → 封顶 | **50 元** |
| 24 小时 | 封顶 | **50 元** |
| 25 小时 | 第一天50元 + 第二天首小时5元 | **55 元** |
| 48 小时 | 第一天50元 + 第二天50元 | **100 元** |

## 六、常见问题

**Q：车牌识别失败，AI 说"未检测到车牌"，怎么办？**

A：识别失败不影响正常使用。直接在车牌号输入框手动输入车牌号码，然后正常选择入场/出场操作执行即可。AI 识别只是辅助手段，不是必须步骤。

---

**Q：入场时提示"该车牌已在场内"是什么意思？**

A：说明该车牌号在数据库中已有一条未出场的记录。可能是：
1. 之前该车已入场但未办理出场手续
2. 车牌号输入有误（请仔细核对）

需要先对该车牌执行出场操作，才能重新入场。

---

**Q：出场后费用显示为 0 元，是系统错误吗？**

A：不是错误，是正常的免费情况。当停车时长在免费时长（默认 15 分钟）以内，系统自动将费用设为 0 并标记为"免费"，直接放行即可，无需用户支付。

---

**Q：支付二维码扫了没有反应？**

A：当前系统为**演示模式**，无需真实扫码。二维码显示后等待约 **10 秒**，系统会自动标记支付成功并关闭弹窗。这是模拟支付流程，实际上不会产生真实资金流转。

---

**Q：普通用户能看到其他人的停车记录吗？**

A：不能。普通用户的查询基于车牌号，只能查看自己输入的车牌号对应的记录。不同车牌号之间的记录完全隔离。

---

**Q：修改了收费规则后，已有的停车记录费用会变化吗？**

A：**不会变化**。规则修改只影响修改之后新产生的出场计费。历史上已计算好的费用不会重新计算，保持原来的金额。

---

**Q：车位管理页面看不到某辆车，但 AI 说入场成功了？**

A：车位管理页面每 30 秒刷新一次，可能还未同步最新数据。点击右上角的**刷新**按钮立即同步，或稍等片刻自动刷新。

---

**Q：管理员账号点击支付按钮提示"权限不足"？**

A：支付功能仅对**普通用户**开放。管理员账号不能执行支付操作，这是系统设计。如需测试支付功能，请使用普通用户账号（如 `user` / `admin123`）登录。

---

**Q：忘记密码但注册时填的手机号也不记得了怎么办？**

A：请联系系统管理员，由管理员在后台直接重置密码。

---

**Q：停车记录中"停车时长"和"费用"列显示为空或 0，正常吗？**

A：正常。车辆**仍在场内（停车中）**时，出场时间为空，停车时长实时计算，费用栏显示为 0（出场后才计费）。出场操作完成后这些字段会自动更新。
